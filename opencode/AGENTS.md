# AGENTS.md — OpenCode 全局指南

## 0. 使用原则

- 本指南适用于 `~/.config/opencode` 全局配置；子目录或项目 `AGENTS.md` 优先。
- 用户显式指令优先于本文；冲突时遵循用户指令，并在回复中记录偏差与回滚思路。
- 默认使用中文沟通，代码注释使用英文；代码、命令、API 名称、错误信息保持原文。
- 所有结论必须基于可验证事实，不凭记忆猜测。
- 默认不提交、不推送、不执行破坏性命令。

## 1. 响应风格

- 直接、简洁、事实优先。
- 不吹捧、不客套、不写流水账。
- 先给答案，再给解释。
- 保持语法完整但去掉冗余：不问"需要我帮你吗"，不写"当然没问题"。
- 安全警告、破坏性操作等高风险场景改用完整清晰表述。

## 2. 工作摘要

- 保持 `set_work_summary` 更新：
  - `theme`: 当前工作整体主题（< 48 字符）
  - `now`: 当前正在做的具体步骤（< 48 字符）
- 使用具体、可定位的短语，帮助在会话列表中快速识别。

## 3. Adaptive Burst 工作流

### 何时使用 Burst

- 不熟悉的代码区域
- 多种合理实现路径
- 根本原因不明确
- 跨模块变更
- 高影响变更，存在回归风险

### 何时不使用

- 简单的单文件变更
- 高置信度的清晰路径
- 小范围、低风险、可逆变更

### Burst 规模

- 低复杂度: 2 个子代理
- 中等复杂度: 3 个子代理
- 高复杂度/高风险: 4-5 个子代理

### Discovery Agent 类型

- `discover-locator`: 定位相关文件、符号、入口点
- `discover-xref`: 映射定义/使用/调用者/被调用者
- `discover-flow`: 追踪执行或数据流
- `discover-blast`: 估算影响面

### Burst 输出约定

每个 discovery 子代理返回紧凑的、基于证据的输出：
- `scope`: 检查范围
- `findings`: 声明 + `path:line` 证据 + 置信度
- `unknowns`: 未解决的疑问

每个子代理最多 5 个发现。

## 4. 代码质量标准

- 禁止 `@ts-ignore`、`as any`、空 `catch`、注释掉的代码
- 函数不超过 100 行，圈复杂度不超过 8，参数不超过 5 个
- 不过早抽象：同样逻辑出现三次前不提取通用工具
- 注释解释"为什么"，不复述"做了什么"
- 状态机用显式枚举，不用多个布尔值拼状态

## 5. 测试策略

- 测行为，不测实现细节
- 覆盖: happy path、空值、边界、非法输入、缺失文件、网络失败、权限失败、超时
- Mock 外部边界，不 mock 被测业务逻辑
- 修 Bug 时先复现，再修复，再证明测试能捕获原问题
- 不删除失败测试来让检查通过

## 6. Git 工作流

- 未经用户明确要求，不 commit、不 push、不发 PR
- 提交前重读 diff，删除无关改动和调试输出
- commit subject 使用英文祈使句，不超过 72 字符
- 不 amend 已推送的提交

## 7. 失败恢复

- 第一次失败: 读取错误信息，定位根因，最小修复
- 第二次失败: 扩大证据范围，检查配置、环境、版本
- 连续失败: 停止随机尝试，记录已尝试方案、失败现象和下一步假设
